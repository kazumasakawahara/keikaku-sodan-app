## UI設計の基本方針

### 設計原則
1. **シンプル・直感的**: 複雑な操作を避け、3クリック以内で目的達成
2. **見やすさ優先**: 大きめのフォント、十分な余白、明確な区切り
3. **効率重視**: よく使う機能はすぐにアクセス可能
4. **エラー防止**: 重要な操作には確認ダイアログ
5. **レスポンシブ**: PC・タブレット両対応

### ターゲットユーザーの特性
- 相談支援専門員（福祉専門職）
- 1日に複数の利用者の記録を入力
- システム操作に慣れていない方も含む
- 情報の検索・閲覧が頻繁

## UI実装のベストプラクティス

### 1. 配色とコントラスト
- テキストと背景のコントラスト比: 4.5:1以上
- 重要なボタンは目立つ色（プライマリカラー）
- 警告・エラーは赤系、成功は緑系

### 2. タイポグラフィ
```
見出し(h1): 24-28px
見出し(h2): 20-24px
見出し(h3): 18-20px
本文: 16px
小さい文字: 14px (多用しない)
```

### 3. スペーシング
- カード間: 20-30px
- セクション間: 40-50px
- フォーム要素間: 15-20px
- 余白は十分に確保

### 4. ボタン
```
主要アクション: [  大きめ・目立つ色  ]
副次アクション: [ 控えめな色・線のみ ]
危険な操作:     [ 赤系・確認ダイアログ ]
```

### 5. フォーム設計
- ラベルは入力欄の上に配置
- 必須項目には*(アスタリスク)
- エラーメッセージは入力欄の直下、赤色で表示
- プレースホルダーで入力例を提示

### 6. アクセシビリティ
- キーボード操作で全機能にアクセス可能
- フォーカス状態を明確に表示
- 画像には代替テキスト
- スクリーンリーダー対応

### 7. フィードバック
- ボタンクリック時: ローディング表示
- 保存成功: 成功メッセージ（緑色）
- エラー発生: エラーメッセージ（赤色）
- 処理中: プログレスバーまたはスピナー

## まとめ

このワイヤーフレームは以下の原則に基づいて設計しています:

✅ **シンプルで直感的** - 複雑な操作を避け、必要な情報にすぐアクセス
✅ **業務フローに沿った設計** - 実際の業務の流れを考慮
